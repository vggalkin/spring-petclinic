# Jenkins Pipeline Documentation

## Описание
Данный Jenkins Pipeline предназначен для автоматизации сборки, тестирования и развертывания Java-приложения (Spring Petclinic). Пайплайн написан на Groovy (Declarative Pipeline) и поддерживает параметризацию, условное выполнение этапов и обработку ошибок.

## Структура Пайплайна

1. **Agent**: Используется `any` (доступный агент на хосте Jenkins).
2. **Tools**: JDK 17 и Maven 3.9.8.
3. **Parameters**:
   - `DEPLOY_ENV`: Выбор окружения (dev, staging, prod).
   - `APP_VERSION`: Версия приложения (строка).
   - `SKIP_TESTS`: Флаг для пропуска тестов (булево).
   - `GIT_BRANCH`: Ветка репозитория для сборки.
4. **Stages**:
   - **Checkout**: Загрузка кода из Git.
   - **Build**: Компиляция проекта (`mvn clean compile`). Включает проверку кода возврата.
   - **Test**: Запуск тестов (`mvn test`). Выполняется только если `SKIP_TESTS` = false.
   - **Deploy**: Развертывание. Выполняется только если ветка `main`.

## Условия выполнения (Conditions)
- Этап **Test** пропускается, если установлен параметр `SKIP_TESTS`.
- Этап **Deploy** выполняется только при сборке ветки `main`.

## Обработка ошибок
- На этапе **Build** используется явная проверка кода возврата команды Maven (`returnStatus: true`).
- В случае ошибки выполняется логирование и принудительное завершение пайплайна с шагом `error`.
- Блок `post` отправляет уведомления по email в случае успеха или неудачи.

## Как запустить
1. Откройте Jenkins в веб-интерфейсе.
2. Создайте новый элемент (New Item) типа **Pipeline**.
3. В настройках сборки выберите **Pipeline script from SCM** (если файл в репо) или **Pipeline script** (вставьте код Jenkinsfile).
4. Нажмите **Build Now**.
5. При запуске вручную (Build with Parameters) вы сможете выбрать значения параметров (версию, окружение, ветку).

## Ожидаемые результаты
- **Успех**: Статус сборки "Success", получено письмо с темой "✅ SUCCESS", артефакты собраны.
- **Неудача**: Статус "Failed", получено письмо с темой "❌ FAILED", в логах консоли указана причина ошибки (например, код возврата Maven).
- **Deploy**: В логах этапа Deploy должно быть сообщение о деплое, если ветка `main`.
